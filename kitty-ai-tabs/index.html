Useful Kitty tab titles for Claude Code and Codex

Show which repository an agent is working in, what it is doing, and when it needs attention.

I often have several terminal tabs running Claude Code and Codex. Default terminal titles do not make it obvious which agent owns a tab, which repository it is changing, or whether it is working, waiting for permission, or finished.

Both coding agents expose lifecycle hooks. Those hooks can call a small script which uses Kitty remote control to update the current tab title. A shell prompt hook restores the normal directory title after the agent exits.

The result

C flurdy.com/docs ⚙ X dotfiles ❓ C tulipan/ABC-123 ✔

A compact title can contain:

  • the agent: C for Claude Code or X for Codex;
  • the Git repository, or current directory outside Git;
  • a shortened branch or task identifier; and
  • a state marker showing what the agent is doing.
State Marker
Session started🌱
Thinking or waiting for input💭
Using a tool⚙
Permission or answer required❓
Turn finished✔
Compacting context🧹

The exact labels and symbols are personal preference. The useful part is that the title changes with the agent lifecycle.

How it works

  1. Kitty listens on a local Unix socket for remote-control commands.
  2. Claude Code or Codex emits a lifecycle hook event.
  3. The hook passes JSON on standard input to a Bash script.
  4. The script derives a short repository and branch label.
  5. kitten @ set-tab-title updates the Kitty tab.
  6. The next Fish prompt restores the ordinary working-directory title.
Keep hook scripts quiet. Agent hooks may capture standard output. Redirect Kitty commands and diagnostics so title updates do not appear in the conversation.

Requirements

  • Kitty terminal;
  • Bash, Git and Python 3;
  • Claude Code, Codex, or both; and
  • optionally Fish for the shell-title reset shown below.

The example uses plain letters and standard Unicode symbols. My own titles also use Nerd Font Git and folder glyphs, but those should only be added when the configured terminal font supports them.

Configure Kitty remote control

Add a local socket to ~/.config/kitty/kitty.conf:

allow_remote_control socket-only
listen_on unix:@kitty

Restart Kitty after changing listen_on. Test the connection from a Kitty shell: 

kitten @ set-tab-title "title test"

If that does not find the socket, specify it explicitly:

kitten @ --to unix:@kitty set-tab-title "title test"
Remote control is powerful. Use a local Unix socket and read Kitty's remote-control security documentation before allowing broader access.

Create a shared title script

Save this portable baseline as ~/.local/bin/kitty-agent-title: 

#!/usr/bin/env bash

AGENT=${1:-agent}
PAYLOAD=$(cat)

json_value() {
  local key="$1"
  printf '%s' "$PAYLOAD" | python3 -c \
    "import json,sys; print(json.load(sys.stdin).get('$key', ''))" \
    2>/dev/null
}

EVENT=$(json_value hook_event_name)
CWD=$(json_value cwd)
[ -z "$CWD" ] && CWD="$PWD"

GIT_ROOT=$(git -C "$CWD" rev-parse --show-toplevel 2>/dev/null || true)
BRANCH=$(git -C "$CWD" branch --show-current 2>/dev/null || true)

if [ -n "$GIT_ROOT" ]; then
  REPO=$(basename "$GIT_ROOT")
else
  REPO=$(basename "$CWD")
fi

case "$BRANCH" in
  main|master|trunk|"") BRANCH="" ;;
  feature/*|fix/*|bugfix/*|chore/*) BRANCH=${BRANCH#*/} ;;
esac

case "$AGENT" in
  claude) LABEL="C $REPO" ;;
  codex)  LABEL="X $REPO" ;;
  *)      LABEL="$AGENT $REPO" ;;
esac

[ -n "$BRANCH" ] && LABEL="$LABEL/$BRANCH"

case "$EVENT" in
  SessionStart)                  MARK="🌱" ;;
  UserPromptSubmit|PostToolUse)  MARK="💭" ;;
  PreToolUse)                    MARK="⚙" ;;
  PermissionRequest)             MARK="❓" ;;
  Stop|SubagentStop)             MARK="✔" ;;
  PreCompact|PostCompact)        MARK="🧹" ;;
  *)                             MARK="" ;;
esac

TITLE="$LABEL"
[ -n "$MARK" ] && TITLE="$TITLE $MARK"

if command -v kitten >/dev/null 2>&1; then
  kitten @ set-tab-title "$TITLE" >/dev/null 2>&1 \
    || kitten @ --to unix:@kitty set-tab-title "$TITLE" >/dev/null 2>&1 \
    || true
fi

# Fallback to terminal title escape sequences.
(
  exec 2>/dev/null
  printf '\e]30;%s\a' "$TITLE" > /dev/tty || true
  printf '\e]2;%s\a' "$TITLE" > /dev/tty || true
) || true

Make it executable:

chmod +x ~/.local/bin/kitty-agent-title

The script deliberately ignores errors. A title failure must never interrupt an agent session.

Connect Claude Code hooks

Merge hooks like these into ~/.claude/settings.json. If that file already has a hooks object, add the event entries rather than replacing the whole file. 

{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.local/bin/kitty-agent-title claude"
          }
        ]
      }
    ],
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.local/bin/kitty-agent-title claude"
          }
        ]
      }
    ],
    "PreToolUse": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "~/.local/bin/kitty-agent-title claude"
          }
        ]
      }
    ],
    "PermissionRequest": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.local/bin/kitty-agent-title claude"
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.local/bin/kitty-agent-title claude"
          }
        ]
      }
    ]
  }
}

Add PostToolUse and PreCompact in the same form if you want the finer state transitions shown earlier.

Connect Codex hooks

Codex reads user hooks from ~/.codex/hooks.json. Like Claude Code, it includes hook_event_name in the JSON payload, so each hook can call the same command: 

{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "~/.local/bin/kitty-agent-title codex",
            "timeout": 5
          }

      }

    "UserPromptSubmit": [

Codex asks you to review new or changed command hooks. Use /hooks in the CLI to inspect and trust the definition. Matching hooks from different configuration layers are combined, so check for duplicates if the title updates more than expected.

Restore the shell title

Without a reset, the completed agent title remains after returning to the shell. Fish can restore a directory-based title whenever it displays a prompt.

Create ~/.config/fish/functions/update_kitty_tab_title.fish: 

function update_kitty_tab_title \
    --description 'Reset Kitty tab title to the shell location' \
    --on-event fish_prompt

    status is-interactive; or return
    set -q KITTY_WINDOW_ID; or return

    set -l title (prompt_pwd)
    test -n "$title"; or set title (pwd)

    if type -q kitten
        kitten @ set-tab-title "$title" >/dev/null 2>&1
        or kitten @ --to unix:@kitty set-tab-title "$title" >/dev/null 2>&1
    end

    printf '\e]30;%s\a' "$title" >/dev/tty 2>/dev/null
    printf '\e]2;%s\a' "$title" >/dev/tty 2>/dev/null
end

Load it from ~/.config/fish/config.fish:

source ~/.config/fish/functions/update_kitty_tab_title.fish

Bash and Zsh can do the same from their prompt hooks. The important detail is that the shell should only reset the title when a prompt is shown, not while the foreground agent is running.

Further refinements

The separate Claude and Codex scripts this article grew from go further than the portable example:

  • resolve the real repository name when using Git worktrees;
  • strip common branch prefixes and shorten ticket branches;
  • show an active or recently completed Beads issue on the main branch;
  • allow a per-session repository alias with KITTY_TITLE_REPO_ALIAS;
  • distinguish subagent and compaction states; and
  • write an optional event log under /tmp for debugging.

Those refinements are useful once the basic lifecycle works, but repository naming and task detection are personal conventions. Keep the first version small enough that a broken title script is easy to diagnose. I keep the surrounding terminal configuration in my dotfiles repository.

Troubleshooting

The title does not change
Test kitten @ --to unix:@kitty set-tab-title test, confirm the script is executable, then inspect the agent's hook configuration.
Codex says the hook needs review
Open /hooks and trust the exact command definition. Editing the hook changes its hash and requires another review.
The hook runs, but uses the wrong repository
Log the received cwd field. Agents may start from a subdirectory, and Git worktrees need extra handling if you want the shared repository name.
The agent prints control output
Redirect both standard output and standard error from every kitten invocation.
Symbols appear as boxes
Use plain ASCII markers or configure a font containing the selected Unicode or Nerd Font glyphs.
